é importante frisar que nessa metodologia é importante que o cliente também esteja disposto para poder sanar possíveis dúvidas.

Aqui já estamos no final da semana e a equipe manda pequenas versões do projeto para que já possa postar e sugerir melhorias.

Quando a equipe simplifica para o cliente o que acontece durante o processo do projeto, para facilitar o entendimento do mesmo.

para que não se desrespeite as premissas do XP, deve-se manter uma tabela de horas de trabalho de no máximo 40h semanais para se ter um ciclo de trabalho saudável com um ambiente com menos estresse.

é assim uma das formas que essa metodologia usa para manter o processo rápido e fácil, todos da equipe tem acesso ao projeto.

são formadas duplas para cada computador, assim garantindo a constante revisão do projeto, assim como duas cabeças pensam melhor que uma, garante o desenvolvimento do projeto e do colaborador em si, além de fortalecer os laços entre os membros da equipe.

padronizar o código tem suas vantagens, além da estética do código parecer ser feito por só uma pessoa, também facilita a manutenção e o entendimento do cod em questão.
