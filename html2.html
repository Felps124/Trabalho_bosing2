COMO FUNCIONA O EXTREME PROGRAMING? Calma, se você ainda tá meio perdido, aqui você encontra uma explicação mais detalhada de como o XP funciona

jogo do planejamento > No início da semana a equipe se encontra com o cliente para definir a melhor estratégia para prosseguir o processo de criação do projeto, aqui é onde se define um escopo que é sempre flexível e negociável

cliente sempre disponível > é importante frisar que nessa metodologia é importante que o cliente também esteja disposto para poder sanar possíveis dúvidas.

pequenos lançamentos ou entregas curtas > Aqui já estamos no final da semana e a equipe manda pequenas versões do projeto para que já possa postar e sugerir melhorias.

Metáfora > Quando a equipe simplifica para o cliente o que acontece durante o processo do projeto, para facilitar o entendimento do mesmo.

design simples > Deve se focar não só em um projeto de fácil desenvolvimento, deve-se pensar se o resultado sairá de forma simples e confortável para o cliente também.

teste de usuários > é o momento que o cliente e os analistas testam as entregas.

ritmo sustentável > para que não se desrespeite as premissas do XP, deve-se manter uma tabela de horas de trabalho de no máximo 40h semanais para se ter um ciclo de trabalho saudável com um ambiente com menos estresse.

propriedade coletiva > é assim uma das formas que essa metodologia usa para manter o processo rápido e fácil, todos da equipe tem acesso ao projeto.

programação pareada > são formadas duplas para cada computador, assim garantindo a constante revisão do projeto, assim como duas cabeças pensam melhor que uma, garante o desenvolvimento do projeto e do colaborador em si, além de fortalecer os laços entre os membros da equipe.

padronização de código > padronizar o código tem suas vantagens, além da estética do código parecer ser feito por só uma pessoa, também facilita a manutenção e o entendimento do cod em questão.

dev orientado a testes > testes unitários antes de colocá-los no código do projeto em si, aumentando assim a eficiência de tempo para cada projeto.

refatoração > é um dos processos que permite a melhoria contínua do projeto, tornando-o mais claro, com menos possibilidade de erros e de duplicação de processos.

integração contínua > toda vez que se criar novas funcionalidades, as mesmas devem ser integradas e testadas imediatamente para que se evite já no começo falhas nessas funcionalidades.
